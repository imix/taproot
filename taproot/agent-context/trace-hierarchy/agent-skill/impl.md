## Design Decisions
- Implemented as an agent skill rather than a CLI command — hierarchy navigation requires reading multiple files dynamically and presenting contextual recommendations ("want to continue with `/tr-implement`?"); a CLI command could only report, not suggest
- Bottom-up is the primary flow (source file → impl → behaviour → intent): this directly satisfies the `agent-context` intent's success criterion "An agent can navigate from a source file up to the behaviour and intent it belongs to"
- `--unlinked` mode delegates to `taproot check-orphans` rather than reimplementing orphan detection — keeps the skill thin and the CLI authoritative for structural checks
- Fuzzy match (step 2 of Main Flow) is attempted only after exact match fails — avoids false positives on partial path overlaps
