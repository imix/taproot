## Design Decisions
- Implemented as an agent skill — structured 5-Why dialogue requires interactive back-and-forth with the developer; no automated heuristic can reach a root cause classification without the developer's knowledge of the codebase
- Reverse lookup for artifact location uses the same Source Files scan pattern as `taproot commithook` — consistent, already proven, no new mechanism needed
- Stop rule for 5-Why is category-based, not iteration-based — the skill stops as soon as a root cause category is assignable; asking unnecessary extra "Why?" rounds wastes context and risks over-engineering the diagnosis
- State transition (complete → needs-rework) is presented to the developer for explicit confirmation before writing — consistent with pause-and-confirm behaviour
- tr-ineed hand-off detection is documented as a pending cross-spec dependency — tr-ineed.md must be updated separately to route bug-shaped language to /tr-bug
