## Design Decisions
- Skill delegates all file-writing to `/taproot:intent` and `/taproot:behaviour` — it is a routing and dialogue layer only, not a document writer.
- Sharpening questions fire before hierarchy search when requirement is too vague to match; this avoids polluting the dedup search with noise.
- When the requirement is too vague to route, the skill delegates to `/tr-grill-me` for structured elicitation — one decision branch at a time, recommended answer first.
- Conversational detection (triggering without explicit `/tr-ineed`) is documented as a Claude Code agent behaviour in the skill's Notes section — it cannot be enforced by the skill file itself.
