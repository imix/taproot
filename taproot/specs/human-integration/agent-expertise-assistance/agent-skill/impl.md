## Design Decisions
- Implemented as a standalone skill (`skills/expertise-help.md`) so it can be invoked directly when a developer is stuck mid-session, and also called inline by other skills at any `[?] Get help` option — same pattern as `/tr-grill-me`
- Skill accepts a `question` input so callers can pass the unanswered question directly; when invoked without arguments the developer is prompted to state the question
- "What the project already shows" section is omitted (not just suppressed) when no relevant project context is found — alternate flow AC-2 requires the absence to be noted explicitly
- Proposal format separates project evidence, draft answer with reasoning, and alternatives into distinct named sections to match the structured output from usecase.md step 5
- When the question is too broad, the skill presents sub-decisions as a numbered list and asks the developer to confirm the order before working through each in sequence — this satisfies AC-4 without re-running the full skill recursively
- External-verification flag is surfaced as a labelled callout ("⚠ Authoritative verification recommended") rather than inline text, so it is visually distinct and the developer cannot miss it — satisfies AC-5
- Skill is added to `SKILL_FILES` (not `MODULE_SKILL_FILES`) because it is developer-callable in any project, not module-scoped
