## Design Decisions
- Implemented as an agent skill rather than a CLI command — structured elicitation requires interactive dialogue with the developer; no automated heuristic can conduct a decision-branch interview
- Agent always provides its recommended answer before the developer responds — transforms the session from a blank-slate interview into a dialogue about recommendations, which is faster and produces better decisions
- "One question at a time" is a hard constraint enforced by the skill wording — batching questions allows vague omnibus answers
- Attribution to Matt Pocock's MIT-licensed grill-me pattern is embedded directly in the skill file and the behaviour spec Notes section
