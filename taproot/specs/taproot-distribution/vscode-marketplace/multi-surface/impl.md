## Design Decisions
- **Extension source in `channels/vscode/`**: all distribution channel artifacts live under `channels/<channel-name>/` (convention). Self-contained directory with its own `package.json`, `tsconfig.json`, and `src/`. Keeps VS Code extension tooling isolated from the root npm workspace.
- **TypeScript compiled to `out/`**: standard VS Code extension convention; `out/extension.js` is the `main` entry point. The `.vscodeignore` excludes `src/` and dev files from the `.vsix`.
- **`taproot.init` opens an integrated terminal**: VS Code's `createTerminal` + `sendText` is the minimal, reliable approach for running a CLI command in the user's workspace. No shell detection, no path resolution — `npx` handles the rest.
- **Publisher `imix-ai`**: registered at `marketplace.visualstudio.com/manage`. Permanent Marketplace URL: `marketplace.visualstudio.com/items?itemName=imix-ai.taproot`.
- **Version kept in sync via CI pre-flight**: `channels/vscode/package.json` version must match the root `package.json` version at release time. The `publish-vscode-extension` CI job checks this before building. The release skill (`skills/release.md`) is responsible for bumping both versions together.
- **Open VSX publish is non-blocking**: `continue-on-error: true` on the Open VSX step — VS Marketplace is the primary channel. A failed Open VSX publish is logged with a manual retry command but does not fail the release.
- **README sync via SHA marker**: `channels/vscode/README.md` embeds a `<!-- root-readme-sha: <sha256> -->` marker. `scripts/sync-channel-readmes.sh --check` compares the embedded SHA against the current `README.md` hash and exits 1 if they differ. The `ci` job runs this check on every push (AC-8).

## Source Files
- `channels/vscode/package.json` — extension manifest: publisher, displayName, description, keywords, categories, command contribution `taproot.init`, Get Started walkthrough definition
- `channels/vscode/package-lock.json` — lockfile generated by `npm install`; required by CI `npm ci`
- `channels/vscode/src/extension.ts` — activation: registers `taproot.init` command; opens integrated terminal and runs `npx @imix-js/taproot init`
- `channels/vscode/tsconfig.json` — TypeScript config targeting CommonJS output in `out/`
- `channels/vscode/.vscodeignore` — excludes `src/`, `node_modules/`, dev config from the `.vsix` package
- `channels/vscode/README.md` — Marketplace listing content; embeds `root-readme-sha` marker for sync check
- `channels/vscode/icon.png` — 128×128 extension icon derived from `assets/logo-512.png`
- `scripts/sync-channel-readmes.sh` — compares embedded SHA in each channel README against root README.md; `--check` mode exits 1 if out of sync
- `.github/workflows/release.yml` — adds `publish-vscode-extension` job with pre-flight version check, `vsce package`, `vsce publish`, `ovsx publish` (non-blocking); `ci` job runs README sync check
- `.github/workflows/test-vscode.yml` — on-demand workflow to build and upload VSIX without publishing
- `taproot/agent/skills/release.md` — step 4 bumps `channels/vscode/package.json` version alongside root `package.json`
